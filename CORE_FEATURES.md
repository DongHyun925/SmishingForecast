## 1. 솔루션 개요

**Smishing Forecast**는 최신 뉴스 데이터를 기반으로 미래에 발생할 수 있는 스미싱 공격을 사전에 예측(Forecast)하고, 자가 진화(Self-Evolution)하는 AI를 통해 방어력을 실시간으로 강화하는 지능형 보안 시스템입니다.

- **Red Team (공격)**: 실제 사회 이슈를 반영한 현실적인 공격 시나리오 생성
- **Blue Team (방어)**: 실시간 탐지 및 취약점 분석 후 즉시 재학습
- **Database (자산화)**: 공격/방어 데이터의 영구적 보존 및 자산화

## 2. 주요 기능 상세 (Feature List)

### 🔴 Red Team: 지능형 공격 시뮬레이션

| 기능명 | 상세 설명 | 해결하는 문제 |
|--------|-----------|---------------|
| **뉴스 기반 시나리오 기획** | 최신 뉴스 기사를 크롤링하여 사회적 이슈(예: 정책 지원금, 택배 파업)를 분석하고, 이를 악용할 수 있는 공격 시나리오를 자동 생성합니다. | 시의성 있는 공격 패턴 예측 불가 문제 해결 |
| **Generative AI 공격 생성** | GPT-4 기반으로 사람의 심리를 교묘하게 파고드는 자연스러운 한국어 스미싱 문자를 생성합니다. (URL/전화번호 없는 스텔스 공격 포함) | 기존 필터를 우회하는 정교한 AI 공격 대응 부재 |
| **다형성(Polymorphism) 공격** | 동일한 의도라도 문구, 어조, 발신자 유형을 다양하게 변형하여 탐지 시스템의 사각지대를 찾아냅니다. | 단일 패턴 방어에만 최적화된 기존 시스템 한계 극복 |

### 🔵 Blue Team: 자가 진화형 방어 시스템

| 기능명 | 상세 설명 | 해결하는 문제 |
|--------|-----------|---------------|
| **RoBERTa 실시간 탐지** | 한국어 특화 모델(klue/roberta-base)을 사용하여 문맥을 파악하고 스미싱 여부를 실시간으로 판별합니다. | 키워드 매칭 방식의 높은 오탐/미탐 문제 해결 |
| **의도 분석 (Intent Analysis)** | 메시지에 담긴 심리적 기제(긴급성, 권위, 이득 유혹 등)와 잠재적 위협 의도를 심층 분석합니다. | 단순 차단을 넘어선 공격 의도 파악 및 대응 전략 수립 |
| **자가 진화 (Self-Evolution)** | 탐지에 실패한 새로운 공격 패턴을 즉시 학습 데이터로 변환하고, 모델을 재학습(Fine-tuning)시켜 방어력을 강화합니다. | 신종 공격 등장 시 대응까지 걸리는 시간(Lead Time) 단축 |

## 3. MVP 핵심 기능 (Minimum Viable Product)

초기 버전(MVP)에서 구현 완료된 핵심 기능은 다음과 같으며, 이는 시스템의 독창성과 가치를 입증하는 최소한의 범위입니다.

### ✅ 구현 완료 (Core Features)

1.  **Context-Aware Attack Generation (맥락 인식 공격 생성)**
    *   [x] 뉴스 데이터 크롤링 및 분석 (`crawler.py`)
    *   [x] 뉴스 내용 기반 공격 시나리오(Intent) 기획 (`planner.py`)
    *   [x] LLM을 활용한 구체적인 스미싱 메시지 생성 (`generator.py`)

2.  **Adaptive Defense System (적응형 방어 시스템)**
    *   [x] RoBERTa 기반 스미싱 탐지 모델 구축 (`detector.py`)
    *   [x] 탐지 확률(Probability) 및 스코어 계산
    *   [x] **핵심 차별점**: 탐지 실패 시 즉시 재학습(Few-shot Learning) 프로세스 구현 (`trainer.py`)

3.  **Integrated Data Pipeline (통합 데이터 파이프라인)**
    *   [x] SQLite 기반 로컬 데이터베이스 구축 (`database_manager.py`)
    *   [x] 뉴스-시나리오-로그-리포트 간 데이터 연계 저장
    *   [x] PDF 리포트 생성 및 다운로드 기능 (`report_generator.py`)

### 🔜 향후 고도화 계획 (Post-MVP)

*   [ ] **다중 모달(Multi-modal) 방어**: 텍스트뿐만 아니라 이미지(QR코드, 캡처) 내 텍스트 인식(OCR) 추가
*   [ ] **연합 학습(Federated Learning)**: 여러 기관의 데이터를 개인정보 유출 없이 공유하여 학습
*   [ ] **API Gateway 구축**: 외부 서비스(금융앱, 통신사)에서 호출 가능한 REST API 제공

## 4. 기능별 해결 과제 및 기대 효과

### 4.1 시의성 있는 위협 예측 (Time-Sensitive Threat forecasting)
*   **문제**: 스미싱 공격은 사회적 이슈(예: 재난지원금, 명절 선물) 발생 직후에 급증하지만, 기존 보안 패치 업데이트는 24~72시간이 소요됩니다.
*   **해결**: 뉴스가 보도되는 즉시 AI가 이를 공격 시나리오로 변환하여 **공격자보다 먼저(Pre-emptive)** 방어 모델을 학습시킵니다.
*   **효과**: **Zero-Day 공격 방어율 상승** 및 대응 시간(Lead Time)을 '일' 단위에서 '분' 단위로 단축.

### 4.2 필터 우회 공격 무력화 (Neutralizing Evasion Attacks)
*   **문제**: 공격자들은 '대출', '광고' 등의 키워드를 피하고, "엄마 나 폰 고장났어"와 같은 일상 대화형(Stealth) 문자를 사용해 차단을 우회합니다.
*   **해결**: 단순 키워드가 아닌 **문맥(Context)과 의도(Intent)**를 분석하는 RoBERTa 모델을 사용하여, 일상 대화 속에 숨겨진 악의적 의도를 탐지합니다.
*   **효과**: **오탐률(False Positive) 감소** 및 지능형 스미싱 탐지 정확도 향상.

### 4.3 보안 데이터의 자산화 (Data Assetization)
*   **문제**: 수많은 스미싱 문자가 수신되지만, 이는 개별적으로 차단되고 삭제되어 패턴 분석을 위한 데이터로 활용되지 못합니다.
*   **해결**: 모든 공격 시도와 방어 결과를 DB에 저장하여, **공격 트렌드 분석** 및 **사후 포렌식**을 위한 귀중한 데이터 자산으로 축적합니다.
*   **효과**: 데이터 기반의 보안 정책 수립 지원 및 **지속적인 AI 성능 고도화** 기반 마련.
